# Signposter Artifacts Reference

## Purpose

Quick reference for local prompt, run, review, and summary artifacts created during the Signposter lifecycle. Raw artifacts stay local. Only bounded summaries and derived comments are posted to GitHub.

## Artifact Directories

### artifacts/prompts/

- Worker prompts for issues:
  `artifacts/prompts/issue-N.md`
- Reviewer prompts for PRs:
  `artifacts/prompts/pr-P-review.md`

### artifacts/runs/

- Worker raw output (full local log):
  `artifacts/runs/issue-N-worker.raw.txt`
- Worker summary (bounded, for report/gate):
  `artifacts/runs/issue-N-worker.summary.md`
- Reviewer raw output:
  `artifacts/runs/pr-P-reviewer.raw.txt`
- Reviewer summary:
  `artifacts/runs/pr-P-reviewer.summary.md`

## Worker Artifacts

**Created by:**
```bash
signposter run --repo $REPO --issue $ISSUE --write-prompt
signposter run --repo $REPO --issue $ISSUE --execute --worktree
```

**Used by:**
```bash
signposter report --repo $REPO --issue $ISSUE --apply
signposter gate --repo $REPO --issue $ISSUE
signposter complete --repo $REPO --issue $ISSUE --apply
```

**Key contents to inspect:**
- `Dirty Guard:` line (must be clean for safe completion)
- `Exit Code:`
- Changed files / scope evidence
- `Scope followed: 100%` (when present)
- Any error or blocker signals

Raw output stays local. The summary is intentionally bounded and safe for `report --apply`.

## Reviewer Artifacts

**Created by:**
```bash
signposter review write-prompt --repo $REPO --pr $PR
signposter review execute --repo $REPO --pr $PR
```

**Used by:**
```bash
signposter review gate --repo $REPO --pr $PR
signposter review submit --repo $REPO --pr $PR --apply
```

Reviewer summaries must contain structured opinion and gate signals. Raw reviewer output stays local. The GitHub review body posted by `review submit --apply` is compact and derived from the gate decision.

## GitHub-Visible Outputs

- Issue report comment: generated by `signposter report --apply`
- GitHub PR review: generated by `signposter review submit --apply`
- Transition comments: claim, complete, fail, integration apply
- Never post full raw logs by default

## Artifact Safety Rules

- Raw logs (`*.raw.txt`) remain local only.
- Do not paste secrets or tokens into GitHub.
- Do not post full OpenClaw / worker output to GitHub.
- Use only bounded summaries and gate-derived text.
- Artifacts are evidence, not authority — gate and review decisions must still pass.
- Keep `artifacts/` out of commits unless intentionally archiving a completed run.

## Common Inspection Commands

```bash
# List artifacts
ls -lah artifacts/prompts/
ls -lah artifacts/runs/

# View bounded worker summary (first ~160 lines)
sed -n '1,160p' artifacts/runs/issue-$ISSUE-worker.summary.md

# View bounded reviewer summary
sed -n '1,160p' artifacts/runs/pr-$PR-reviewer.summary.md

# Extract key signals across runs
grep -n "Dirty Guard\|Exit Code\|Scope\|Verdict\|Confidence\|NEEDS-WORK\|PASS" artifacts/runs/*.md
```

## Recovery Notes

- Missing worker summary:
  ```bash
  signposter run --repo $REPO --issue $ISSUE --execute --worktree
  ```

- Missing reviewer summary:
  ```bash
  signposter review execute --repo $REPO --pr $PR
  ```

- Bad or unclear summary:
  Inspect the corresponding `.raw.txt` file locally first.

- Gate NEEDS-WORK:
  Inspect summary + raw for the issue, improve evidence or rerun worker, then re-report and re-gate.

- Reviewer gate blocked:
  Inspect reviewer summary and the PR diff locally before resubmitting.
