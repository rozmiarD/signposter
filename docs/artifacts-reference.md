# Signposter Artifacts Reference

## Purpose

Quick reference for local prompt, run, review, and summary artifacts created during the Signposter lifecycle. Raw artifacts stay local. Only bounded summaries and derived comments are posted to GitHub.

## Artifact Directories

### artifacts/prompts/

- Worker prompts for issues:
  `artifacts/prompts/issue-N.md`
- Reviewer prompts for PRs:
  `artifacts/prompts/pr-P-review.md`

### artifacts/runs/

- Worker raw output (full local log):
  `artifacts/runs/issue-N-worker.raw.txt`
- Worker summary (bounded, for report/gate):
  `artifacts/runs/issue-N-worker.summary.md`
- Preserved worker runtime diagnostics after takeover:
  `artifacts/runs/issue-N-worker.codex-runtime.raw.txt`
  `artifacts/runs/issue-N-worker.codex-runtime.summary.md`
- Reviewer raw output:
  `artifacts/runs/pr-P-reviewer.raw.txt`
- Reviewer summary:
  `artifacts/runs/pr-P-reviewer.summary.md`
- Preserved reviewer runtime diagnostics after takeover:
  `artifacts/runs/pr-P-reviewer.codex-runtime.raw.txt`
  `artifacts/runs/pr-P-reviewer.codex-runtime.summary.md`

## Worker Artifacts

**Created by:**
```bash
signposter run --repo $REPO --issue $ISSUE --write-prompt
signposter run --repo $REPO --issue $ISSUE --execute --worktree
```

**Used by:**
```bash
signposter report --repo $REPO --issue $ISSUE --apply
signposter gate --repo $REPO --issue $ISSUE
signposter complete --repo $REPO --issue $ISSUE --apply
```

**Key contents to inspect:**
- `Dirty Guard:` line (must be clean for safe completion)
- `Exit Code:`
- Changed files / scope evidence
- `Scope followed: 100%` (when present)
- Any error or blocker signals

Raw output stays local. The summary is intentionally bounded and safe for `report --apply`.

**Canonical worker summary fields:**
- `Repository`
- `Issue`
- `Agent`
- `Exit Code`
- `Dirty Guard`
- `Task execution complete`
- `Acceptance`
- scoped completion evidence
- files changed
- targeted validation evidence
- full validation evidence
- safety notes
- gate recommendation

For documentation-only changes, the summary must also make the docs-only scope
explicit:

- `Docs-only scope: yes`
- `Changed files are documentation-only: yes`
- `Code behavior unchanged: yes`
- `Scope stayed inside requested documentation task: yes`
- `Dirty guard: clean`

Manual worker summaries generated by `signposter artifact write-worker-summary`
also include structured validation records:

```text
Schema: signposter.validation-result.v1
Fields: scope, status, command
```

These records let later review and gate steps cite exactly which targeted and
full validation commands were used.

## Reviewer Artifacts

**Created by:**
```bash
signposter review write-prompt --repo $REPO --pr $PR
signposter review execute --repo $REPO --pr $PR
```

**Used by:**
```bash
signposter review gate --repo $REPO --pr $PR
signposter review submit --repo $REPO --pr $PR --apply
```

Reviewer summaries must contain structured opinion and gate signals. Raw reviewer output stays local. The GitHub review body posted by `review submit --apply` is compact and derived from the gate decision.

**Canonical reviewer summary fields:**
- `Agent`
- `PR`
- `Verdict: APPROVE | NEEDS_CHANGES | BLOCK`
- `Confidence: <float>`
- `Risk: low | medium | high`
- `Scope match: yes`
- `CI considered: yes`
- `Merge recommendation: yes`
- `Automerge eligible: yes | no`
- `Findings`
- `Reasoning summary`
- validation considered
- validation provenance
- safety notes

Reviewer artifacts do not submit a GitHub review by themselves. They are local
evidence consumed by `review gate` and `review submit --apply`.

## GitHub-Visible Outputs

- Issue report comment: generated by `signposter report --apply`
- GitHub PR review: generated by `signposter review submit --apply`
- Transition comments: claim, complete, fail, integration apply
- Never post full raw logs by default

## Artifact Safety Rules

- Raw logs (`*.raw.txt`) remain local only.
- Do not paste secrets or tokens into GitHub.
- Do not post full OpenClaw / worker output to GitHub.
- Use only bounded summaries and gate-derived text.
- Artifacts are evidence, not authority — gate and review decisions must still pass.
- Keep `artifacts/` out of commits unless intentionally archiving a completed run.
- Canonical summaries (`issue-N-worker.summary.md`,
  `pr-P-reviewer.summary.md`) must be parser-compatible and gate-friendly.
- Runtime/backend diagnostics that contain failure markers must be preserved
  under diagnostic names such as `.codex-runtime.*`, then replaced by a clean
  canonical manual summary after inspection.
- GitHub-facing comments may mention local artifact paths, but must not include
  unbounded raw logs, secrets, provider tokens, or noisy backend output.

## Takeover Preservation Contract

When an execution backend cannot produce usable output, preserve the raw and
summary diagnostics before writing the canonical bounded summary.

Worker takeover:

```bash
cp artifacts/runs/issue-$ISSUE-worker.raw.txt \
  artifacts/runs/issue-$ISSUE-worker.codex-runtime.raw.txt
cp artifacts/runs/issue-$ISSUE-worker.summary.md \
  artifacts/runs/issue-$ISSUE-worker.codex-runtime.summary.md
```

Reviewer takeover:

```bash
cp artifacts/runs/pr-$PR-reviewer.raw.txt \
  artifacts/runs/pr-$PR-reviewer.codex-runtime.raw.txt
cp artifacts/runs/pr-$PR-reviewer.summary.md \
  artifacts/runs/pr-$PR-reviewer.codex-runtime.summary.md
```

After preservation, inspect the worktree, PR, and diagnostics locally. Continue
only with a canonical summary that records scoped completion or review evidence
without reusing unsafe runtime markers as gate evidence.

## Redaction Boundary

Signposter comments and reviews are derived from bounded summaries, not raw
execution logs. Before content can reach GitHub it must remain within these
boundaries:

- allowed: artifact path, scoped summary, validation command names, gate result,
  review verdict, compact safety notes;
- forbidden: secrets, provider tokens, complete raw logs, long backend output,
  unrelated environment details, and misleading ready/apply wording;
- required: clear statement when no GitHub mutation, issue close, merge, or
  cleanup happened.

If a summary needs details from a noisy backend run, summarize the operator
finding in neutral language and keep the raw diagnostic artifact local.

## Common Inspection Commands

```bash
# List artifacts
ls -lah artifacts/prompts/
ls -lah artifacts/runs/

# View bounded worker summary (first ~160 lines)
sed -n '1,160p' artifacts/runs/issue-$ISSUE-worker.summary.md

# View bounded reviewer summary
sed -n '1,160p' artifacts/runs/pr-$PR-reviewer.summary.md

# Extract key signals across runs
grep -n "Dirty Guard\|Exit Code\|Scope\|Verdict\|Confidence\|NEEDS-WORK\|PASS" artifacts/runs/*.md
```

## Recovery Notes

- Missing worker summary:
  ```bash
  signposter run --repo $REPO --issue $ISSUE --execute --worktree
  ```

- Missing reviewer summary:
  ```bash
  signposter review execute --repo $REPO --pr $PR
  ```

- Bad or unclear summary:
  Inspect the corresponding `.raw.txt` file locally first.

- Gate NEEDS-WORK:
  Inspect summary + raw for the issue, improve evidence or rerun worker, then re-report and re-gate.

- Reviewer gate blocked:
  Inspect reviewer summary and the PR diff locally before resubmitting.
